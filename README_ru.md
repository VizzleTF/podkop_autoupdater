# podkop_updater

Telegram-бот для роутеров OpenWrt/ImmortalWrt, который следит за релизами
[podkop](https://github.com/itdoginfo/podkop) и позволяет запускать
обновление и перезагрузку прямо из чата. Реализован как единый статический
Go-бинарь, управляемый procd.

## Возможности
- **Дашборд-меню**: одно отслеживаемое сообщение показывает карточку статуса
  (версии podkop и updater, DNS, активный тир транспорта, время последней
  проверки) с кнопками: проверить, перезагрузить podkop, открыть подменю
  Бэкапы / Настройки. Контекстный ряд «⬆️ Обновить» появляется при наличии
  обновления.
- Также работают slash-команды: `/menu`, `/check_podkop`, `/check_self`,
  `/check_dns`, `/restart`, `/status`, `/log`
- **Периодическая проверка** (по умолчанию каждые 6 часов) **и** podkop, **и**
  самого updater; новая версия шлёт свежее уведомление.
- **Откат версии podkop**: на любой поддерживаемый релиз (≥ 0.7.0) — ставится
  прямо из пакетов того релиза. Версии с сохранённым бэкапом конфига
  предлагаются первыми; откат в один тап предлагается автоматически, если
  обновление уронило DNS.
- **Бэкап / восстановление конфига** (📦 Бэкапы): версионированные снимки
  `/etc/config/podkop` с датой-временем — создать, восстановить (версия →
  дата/время), удалить, с авто-ретеншеном. При откате конфиг целевой версии
  восстанавливается автоматически.
- **Меню настроек** (⚙️ Настройки), применяется на лету без рестарта:
  раздельные тумблеры авто-обновления podkop и updater, интервал проверки,
  ретеншен бэкапов, форс обновления emergency-IP, правка имени роутера и
  списка админов, показ текущего конфига (токен скрыт).
- **Контроль доступа**: ограничить команды конкретными Telegram user ID;
  **guard от двойного клика** не даёт запустить два `install.sh` от root
  одновременно.
- **Защита цепочки поставки**: `install.sh` подкопа качается с пином на тег
  релиза (не HEAD ветки); self-update сверяет бинарь с опубликованным
  `.sha256`.
- **3-уровневый транспорт**: SOCKS5 через podkop → прямой → аварийные IP
  Telegram, со sticky последним рабочим тиром, который периодически
  сбрасывается на основной путь. Аварийные IP обновляются раз в сутки через
  DoH (параллельно) и пишутся в UCI, переживая перезагрузку.
- **Атомарное self-update** с `.bak` бэкапом; procd respawn'ит в новый бинарь.
  Проверка DNS после restart/update/отката поллит `fakeip.podkop.fyi`; провал
  выводится явно.

## Требования
- Роутер OpenWrt или ImmortalWrt (поддерживаемые архитектуры: amd64, arm64,
  armv7, mipsle softfloat, mips softfloat)
- Токен Telegram-бота (от [@BotFather](https://t.me/BotFather)) и chat_id
  (от [@get_id_bot](https://t.me/get_id_bot))

## Установка

```sh
sh -c "$(curl -sfL https://raw.githubusercontent.com/VizzleTF/podkop_autoupdater/main/install.sh)"
```

Скрипт определяет архитектуру, останавливает старую версию (bash или Go),
качает подходящий бинарь из последнего релиза GitHub, ставит procd-сервис
и спрашивает токен/chat_id, если их ещё нет в UCI.

Если на роутере есть собственный бот подкопа `/usr/bin/podkop_bot` —
остановите и отключите его. Два демона на один и тот же токен будут
воровать друг у друга обновления.

## Конфигурация

Настройки хранятся в UCI (`/etc/config/podkop_updater`):

```sh
uci set podkop_updater.settings.bot_token="ВАШ_ТОКЕН"
uci set podkop_updater.settings.chat_id="ВАШ_CHAT_ID"
uci set podkop_updater.settings.check_interval=6   # часы
uci set podkop_updater.settings.router_label="Дом"  # опционально: показывается в шапке
uci set podkop_updater.settings.admin_ids="123456789 987654321"  # опционально: разрешённые user ID
uci set podkop_updater.settings.auto_update=1        # опц.: авто-установка релизов podkop
uci set podkop_updater.settings.auto_update_self=1   # опц.: авто-установка релизов updater
uci set podkop_updater.settings.backup_keep=10       # опц.: хранить N бэкапов конфига (0 = без лимита)
uci commit podkop_updater
```

Большинство из этих параметров также правятся на лету из меню **⚙️ Настройки**
в чате (изменения пишутся в UCI и применяются сразу). `bot_token` и `chat_id`
задаются только тут или через установщик.

| ключ | значение |
|------|----------|
| `router_label` | Имя жирным в шапке каждого сообщения; различает роутеры в общем чате. Пусто = hostname. |
| `admin_ids` | Список Telegram user ID через пробел. Если задан — команды только им (проверка `From.ID`), остальным «Нет доступа». Пусто = любой в чате. |
| `auto_update` | `1` = периодическая проверка сама ставит новые релизы **podkop**. |
| `auto_update_self` | `1` = сама ставит и новые релизы **updater** (с проверкой sha256). По умолчанию выкл, чтобы плохой self-релиз не сломал бота молча. |
| `backup_keep` | Сколько бэкапов конфига хранить; старые удаляются первыми. `0` = без лимита. |

Демон сам пишет обнаруженные аварийные IP в
`podkop_updater.settings.emergency_ips` (через пробел) и хранит id
сообщения-меню в `podkop_updater.settings.menu_mid`. Бэкапы конфига лежат
рядом с живым конфигом как `/etc/config/podkop.bak-<версия>-<дата-время>`
(точка в имени не даёт UCI их подхватить).

## Управление сервисом

```sh
/etc/init.d/podkop_updater start
/etc/init.d/podkop_updater stop
/etc/init.d/podkop_updater restart
```

Логи: `/tmp/podkop_update.log` (in-place ротация при ~200 строках).
